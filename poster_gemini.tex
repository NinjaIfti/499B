\begin{column}{\colwidth}

  \begin{block}{Introduction}

    Recorded lectures pack speech, slides and board work into a single video,
    but that content stays \textbf{locked and unsearchable}. We present
    \textbf{LectureAssist}, a multimodal agentic system that extracts, indexes
    and reasons over a lecture to produce grounded, adaptive study material
    at \textbf{zero API cost} on a free Colab T4 GPU.

  \end{block}

  \begin{block}{Motivation \& Problem}

    Students re-watch hours of recorded lectures to find a single concept, and
    have no fast way to self-test on what the teacher actually emphasised.

    \begin{itemize}
      \item Lecture knowledge is locked inside \textbf{video, speech and slides},
        rarely available as clean text.
      \item Generic quiz tools \textbf{hallucinate} off-topic, ungrounded questions.
      \item One-size-fits-all difficulty ignores \textbf{each student's weak spots}.
    \end{itemize}

    \textbf{Goal.} An offline-capable system that \emph{reads the board, hears the
    lecture}, and produces \emph{grounded, validated and adaptive} study tools at
    \textbf{zero API cost}.

  \end{block}

  \begin{alertblock}{Key Contributions \& Why It's Better}

    \begin{itemize}
      \item \textbf{Validation-gated, one-at-a-time quiz generation} that
        eliminates hallucinated, off-topic and duplicate questions,
        \emph{better than} generic LLM quiz tools that emit a whole batch
        with no per-question check.
      \item \textbf{Fully local, \$0-API} multi-agent pipeline on a single free
        T4 GPU (Ollama + Gemma~3), \emph{better than} cloud-API tools that
        incur per-call cost and require internet access.
      \item \textbf{Multimodal extraction} with per-surface OCR preprocessing
        for slides, blackboards and whiteboards, \emph{better than}
        plain-transcript tools that miss board/slide content entirely.
      \item \textbf{Adaptive learning} grounded with clickable lecture
        \textbf{timestamps}, \emph{better than} static quiz banks that
        ignore each student's weak topics and cite no source.
    \end{itemize}

  \end{alertblock}

  \begin{block}{End-to-End Pipeline}

    A lecture video or document flows left-to-right through extraction, a grounded
    knowledge base, the agent orchestrator, and finally validated study outputs.

  \begin{block}{Methodology}

    \heading{1. Multimodal Extraction}

    Every modality is recovered and time-aligned \emph{before} any LLM call.

    \begin{itemize}
      \item \textbf{Visual / OCR (EasyOCR).} Frames sampled $\sim$5\,s apart,
        \emph{classified} (slide / blackboard / whiteboard / scene) with per-class
        preprocessing (CLAHE, sharpening, Otsu, chalk inversion). \textbf{EN}.
      \item \textbf{Speech (Faster-Whisper).} GPU \texttt{float16} / CPU
        \texttt{int8} fallback, per-segment timestamps.
      \item \textbf{Documents.} PyMuPDF, python-docx, python-pptx (structure kept).
      \item \textbf{Drive OCR cache} keyed by video hash, re-uploads skip re-scan.
    \end{itemize}

    \heading{2. Multi-Agent Orchestration}

    A central \textbf{Orchestrator} runs specialist agents; independent agents run
    \textbf{concurrently}. Each follows
    \texttt{should\_run $\to$ plan $\to$ execute $\to$ validate $\to$ retry}, and
    any crash is caught and logged so the pipeline \textbf{degrades gracefully}.

    \heading{3. Grounded Retrieval (RAG)}

    Speech and board/slide text are embedded with \textbf{Sentence-Transformers}
    (\texttt{MiniLM}) and indexed in \textbf{FAISS} (cosine). Every quiz, summary
    and chat answer retrieves \textbf{top-$k$} chunks first, anchored to real
    content with a clickable timestamp.

      \caption{Retrieval path: every generated answer is grounded before it reaches the student.}
    \end{figure}

\begin{column}{\colwidth}

  \begin{block}{Agentic Quiz Generation}

    Questions are built \textbf{one at a time} and must \emph{pass validation}
    before the next is generated, eliminating hallucinated, off-topic or
    duplicate questions. Modes: \emph{Auto}, \emph{Manual} (Easy / Med /
    Hard), \emph{Adaptive} (targets weak topics).

    \begin{figure}
      \centering
      \begin{tikzpicture}[node distance=1.6cm and 1.6cm]
        \node[laproc, text width=8cm] (plan){\textbf{Planner}\\{\normalfont\small topics, mix}};
        \node[lallm, right=of plan, text width=8cm] (gen){\textbf{Generator}\\{\normalfont\small ONE question}};
        \node[laval, right=of gen, text width=9cm] (val){\textbf{Validator}\\{\normalfont\small grounded, difficulty, unique}};
        \node[lallm, below=1.9cm of gen, text width=8cm] (refine){\textbf{Refiner}\\{\normalfont\small targeted fix}};
        \node[laout, below=1.9cm of val, text width=8cm] (pass){\textbf{Accept}\\{\normalfont\small next slot}};
        \draw[laarr](plan)--(gen);
        \draw[laarr](gen)--(val);
        \draw[laarrg](val)--(pass) node[lalbl, midway, right]{PASS};
        \draw[laloop](val)--(refine) node[lalbl, midway, above]{FAIL};
        \draw[laloop](refine)--(gen) node[lalbl, midway, left]{retry $\le 3$};
      \end{tikzpicture}
      \caption{Generate $\to$ validate $\to$ refine loop ($\le 3$ retries per slot).}
    \end{figure}

    \textbf{Validation gate:} grounded in lecture, correct difficulty, follows
    instructions, not a duplicate ($>$85\% similarity rejected).

  \end{block}

  \begin{block}{Adaptive Learning}

    \begin{itemize}
      \item \textbf{Self-improving difficulty.} A Difficulty Adapter tracks
        per-topic accuracy, flags weak topics ($<$50\%) and steers the planner there.
      \item \textbf{AI short-answer grading} with partial credit and feedback.
    \end{itemize}

  \end{block}
