# LectureForge — What We Found About the Evaluation

This documents what the agentic-vs-non-agentic quiz evaluation actually revealed
during testing on `EEE_452_Lecture_1.pdf`, the issues it surfaced, and how to
present the results. All numbers below are from real runs (LLM-as-judge with the
larger `gemma3:12b` judging questions generated by `gemma3:4b`).

## Setup

- **Agentic pipeline** — per-question Plan → Generate → Validate → Retry, with a
  self-validator, RAG grounding tool-use, and validator-chosen remediation.
- **Non-agentic baseline** — a single LLM call writes one question on the same
  topic, no validation, no retry.
- **Judge** — independent larger model (`gemma3:12b`) scores each question 1–5 on
  correctness, clarity, difficulty-match, and overall, and accepts/rejects it.
- **Metrics beyond the judge** — duplicate rate, semantic grounding and
  inter-question diversity (sentence-embedding based), structural validity, and
  the validator-vs-judge F1 (how well the agent's own validator agrees with the
  independent judge).

## Key findings

### 1. The LLM judge alone is not enough — it saturates and ties
On medium difficulty the judge gave **5/5 to almost every question on both
sides**, so all three segments came out as ties (agentic 5.0 vs non-agentic 5.0).
A single judge score cannot, on its own, separate the two pipelines. This is the
central reason the evaluation needs additional, deterministic metrics.

*Fixes applied:* (a) the judge was re-calibrated to use the full 1–5 range
(reserve 5 for excellent, penalise generic/duplicate framing); (b) a
deterministic tiebreaker (grounding − duplicate rate) decides a segment when the
judge scores tie.

### 2. Sample size matters — F1 is meaningless at n=1
With one question, validator-vs-judge F1 can only be 0 or 1, and early single
question runs showed F1 = 0 from a single disagreement. At **n = 10** the F1
became interpretable (see below). **Conclusion: run ≥ 10 questions per type for
any reportable number.**

### 3. Medium difficulty (10 MCQ) — both look equal to the judge, but differ on metrics
- Judge segments: **tie** (agentic 5.0 vs non-agentic 5.0 on overall, in-context,
  difficulty).
- Accept rate: agentic 100%, non-agentic 100%.
- Semantic grounding: agentic 0.569, non-agentic 0.637.
- **Duplicate rate: agentic 30% vs non-agentic 40%** — agentic repeats itself
  less.
- **Validator-vs-judge F1 = 0.571 (precision 1.0, recall 0.4).**

Reading: the judge sees both as equally good, but the agentic pipeline is less
repetitive while staying comparably grounded. The validator never approved a
question the judge would reject (precision 1.0) but rejected 60% of questions the
judge would accept (recall 0.4) — i.e. **the agent's self-validator is
conservative / over-strict, not lax.**

### 4. Hard difficulty (5 MCQ) — the most revealing run
- Judge segments: **non-agentic won all three** (overall 5.0 vs 3.6, in-context
  5.0 vs 3.4, difficulty 4.0 vs 3.2).
- Accept rate: agentic 60%, non-agentic 100%.
- Semantic grounding: agentic 0.453, non-agentic 0.612.
- **Duplicate rate: agentic 0% vs non-agentic 80%.**

What actually happened:
- The **agentic** pipeline, pushed toward "hard", generated questions on **Net
  Present Value / present-value calculations** — concepts **not present in the
  lecture** (which covers willingness-to-pay, consumer/producer surplus, wealth
  creation). The judge correctly rejected them as ungrounded, dragging the
  agentic average down.
- The **non-agentic** baseline scored 5/5 by repeatedly asking essentially the
  **same safe question** ("the central argument of Lecture 1 regarding wealth
  creation") — hence its **80% duplicate rate**.

Reading: **the LLM judge alone would crown the repetitive baseline as the
winner.** The duplicate metric (0% vs 80%) shows that "win" is hollow — the
baseline is looping one question. The agentic pipeline is far more diverse but
overreached on grounding at hard difficulty.

*Fix applied:* the agentic generator is now constrained to keep questions
answerable from the lecture only ("make it harder by deeper reasoning on the same
material, never by importing outside topics"), to stop the NPV-style
hallucinations.

### 5. The real discriminators are diversity and grounding, not the raw judge score
Across runs, the judge score alone was either tied or favoured the safe baseline.
The metrics that consistently distinguished the pipelines were **duplicate rate**
(agentic far lower) and **grounding** (a genuine agentic weakness at hard
difficulty). This is the core argument for a multi-metric evaluation.

## Metric reference

| Metric | Meaning | Better |
|---|---|---|
| overall / correctness / difficulty-match (judge) | judge quality 1–5 | higher |
| accept rate | fraction the judge would put on a real exam | higher |
| duplicate rate | near-identical questions (difflib > 0.85) | lower |
| semantic grounding | question↔lecture embedding similarity | higher |
| inter-question similarity → diversity | repetition across the set | lower sim |
| validator-vs-judge F1 (agentic only) | agreement of the agent's own validator with the independent judge | higher |

## Issues found and fixed during evaluation

1. **Non-agentic generation returned nothing** (4B failed JSON) → fell back to the
   larger model so a baseline question is always produced.
2. **Judge saw only 1200 chars** of the lecture → falsely flagged grounded
   questions as "not in the text" → widened to ~4500 chars.
3. **Internal validator saw only 700 chars** → over-strict grounding → widened to
   ~2500 chars (also reduces false retries).
4. **Judge defaulted everything to 5/5** → recalibrated to the full range +
   deterministic tiebreaker so segments produce real winners.
5. **Agentic hard questions hallucinated out-of-lecture content (NPV)** → added a
   grounding rule to the generator prompt.

## How to present this

- Don't lead with the raw judge score — on its own it ties or favours the
  repetitive baseline.
- Lead with the **trade-off**: the agentic pipeline produces **far more diverse**
  questions (0–30% duplicates vs 40–80% for non-agentic) and its self-validator is
  **conservative** (precision 1.0), while the baseline scores well by **repeating
  one safe question**.
- Use **n ≥ 10** for any quoted number; mark single-question runs as illustrative.

## Limitations

- The judge is an LLM, not a human — scores are a relative signal, not ground
  truth.
- Generator (`gemma3:4b`) and judge (`gemma3:12b`) share the Gemma-3 family; the
  larger-model-as-judge choice reduces but does not remove shared bias.
- Results are from a single lecture; broader claims need multiple lectures and
  more repeats.
